# Swarm lane 04: Anatomist

## Overview

Lane 04 (Anatomist) is responsible for modeling vertebra positioning and skeletal articulation using validated measurements from lane 03 (Validator). This agent processes validation reports and produces a 3D skeletal rig that defines joint positions and rotation constraints for the dinosaur skeleton.

## Input Contract

The Anatomist receives a `validation_report` from lane 03 containing:
- Validated vertebral measurements and dimensions
- Articulation point coordinates
- Confidence scores for each measurement
- Skeletal segment parameters

## Processing Pipeline

The Anatomist performs the following steps:

1. **Parse Validation Report**: Extract validated measurements and articulation data from the input validation report.
2. **Compute Joint Positions**: Calculate 3D positions for each vertebra and skeletal joint based on validated measurements and anatomical constraints.
3. **Define Rotation Constraints**: Establish rotation limits and degrees of freedom for each articulation point based on paleontological literature and biomechanical principles.
4. **Build Skeletal Rig**: Construct a hierarchical skeletal structure referencing the 3D layout and segment definitions from `dinolab/web/src/data/skeleton3dLayout.ts` and `skeleton3dSegments.ts`.
5. **Validate Rig Integrity**: Ensure the generated rig maintains anatomical plausibility and joint coherence.

## Output Contract

The Anatomist produces a `rig_model` containing:
- **Joint Positions**: 3D coordinates for each vertebra and articulation point
- **Rotation Constraints**: Euler angle limits and degrees of freedom per joint
- **Segment Hierarchy**: Parent-child relationships defining the skeletal chain
- **Metadata**: Source validation report reference, anatomist version, and processing timestamp

## Integration Points

- **Upstream**: Consumes validated skeletal measurements from lane 03 (Validator)
- **Downstream**: Produces rig model consumed by lane 06 (Technician) for rendering pipeline
- **Data References**: Uses skeleton layout and segment definitions from `dinolab/web/src/data/`

## Key Responsibilities

The Anatomist agent ensures that:
- All joint positions respect validated measurements with appropriate tolerances
- Rotation constraints prevent anatomically impossible poses
- The skeletal hierarchy maintains proper parent-child relationships
- Output rig is compatible with the rendering pipeline's expectations
- Articulation points align with paleontological understanding of dinosaur skeletal mechanics
