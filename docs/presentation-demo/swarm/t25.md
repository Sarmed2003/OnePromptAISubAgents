# Swarm lane 25

## Volunteer Agent: Access Control Tagging

Lane 25 is responsible for tagging specimens with access control metadata to determine their visibility and public accessibility. This lane operates as a volunteer agent that processes specimen information from upstream lanes and generates access control tags based on provenance data (from lane 15) and preservation status (from lane 22).

## Core Responsibilities

The volunteer agent in lane 25 performs the following key functions:

- **Access Classification**: Analyzes provenance information and preservation status to determine appropriate visibility levels for each specimen.
- **Tag Generation**: Creates `access_control_tags` containing visibility levels that can be one of: `public`, `restricted`, or `internal`.
- **Data Persistence**: Stores generated tags in DynamoDB (lane 10) for reliable access and retrieval.
- **Downstream Integration**: Provides tagged specimen data to consuming lanes including lane 30 (administrator) for permission enforcement and lane 28 (manager) for tour access management.

## Visibility Levels

- **Public**: Specimens that meet all criteria for unrestricted public access and have appropriate preservation status.
- **Restricted**: Specimens with limited access due to provenance concerns or preservation requirements.
- **Internal**: Specimens that are restricted to internal use only and not available for public or tour access.

## Data Flow

The lane 25 volunteer agent consumes processed specimen data from lane 15 (provenance information) and lane 22 (preservation status), applies classification logic, and outputs tagged specimens to DynamoDB. These tags are subsequently consumed by downstream lanes for access control enforcement and tour management purposes.
