# Swarm lane 08

## Engineer Agent: Bedrock API Client Integration

Lane 08 is responsible for integrating the Bedrock API client to enable question-and-answer functionality on bone anatomy and fossil data. This lane bridges the user interface specification from lane 07 with a fully functional, API-ready service that leverages AWS Bedrock's Converse API for intelligent responses.

## Architecture Overview

The engineer agent in lane 08 operates at the intersection of two critical components:

- **Input Contract**: Receives `ui_specification` from lane 07, which defines the user-facing interface requirements and data models for bone anatomy and fossil queries.
- **Output Contract**: Produces `api_ready_service`, which is a fully integrated Bedrock API client with proper bindings, error handling, and response formatting.

## Core Responsibilities

### 1. Bedrock API Client Integration
The engineer agent integrates the existing `oneprompt/llm_client.py` module, which provides the foundational LLM client abstraction. This client is enhanced with specific Bedrock Converse API bindings to support:

- Multi-turn Q&A conversations about bone anatomy
- Fossil data retrieval and contextual analysis
- Structured response formatting for downstream consumption
- Proper credential management and authentication

### 2. Service Implementation
The primary service implementation resides in `dinolab/infra/functions/ask/app.py`. This Lambda function handler orchestrates:

- Request validation against the UI specification schema
- Bedrock API invocation with appropriate prompting strategies
- Response transformation and formatting
- Error handling and graceful degradation
- Logging and observability hooks

### 3. API Integration Specification
The output artifact `api_integration_spec` documents:

- Bedrock Converse API endpoint bindings
- Request/response payload structures
- Model selection and configuration parameters
- Rate limiting and timeout policies
- Error codes and recovery strategies

## Downstream Dependencies

- **Lane 09 (QA)**: Receives the `api_ready_service` for comprehensive testing, including happy-path Q&A flows, edge cases with malformed bone anatomy queries, and fossil data accuracy validation.
- **Lane 11 (Security)**: Conducts authentication audit of the Bedrock API integration, verifying IAM role bindings, credential rotation policies, and input sanitization for prompt injection prevention.

## Implementation Status

The engineer agent ensures that all Bedrock API bindings are production-ready, with proper error handling, logging, and monitoring instrumentation in place before handoff to QA and security lanes.
