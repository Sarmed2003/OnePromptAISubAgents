# Swarm lane 07

## Designer Agent: UI Layer for Bone Selection and Inspection

Lane 07 is responsible for creating the interactive user interface layer that enables users to select and inspect individual bones from the 3D rendered asset. This designer agent consumes the `render_ready_asset` output from lane 06 and produces a comprehensive `ui_specification` that defines bone selection interactions, detail panels, and layer toggle controls.

## Input Contract

This lane receives the following input from lane 06:
- **render_ready_asset**: A fully prepared 3D asset with bone hierarchy, materials, and textures ready for rendering

## Output Contract

This lane produces the following output for consumption by lane 08:
- **ui_specification**: A complete UI specification containing:
  - Interactive bone selection mechanism with visual feedback
  - Detail panels for displaying bone properties and metadata
  - Layer toggle controls for showing/hiding bone groups
  - Event handlers and state management patterns

## Key Responsibilities

### Bone Selection UI
The designer agent extends `BoneDetailPanel.tsx` to create an interactive component that allows users to click on bones in the 3D view and see detailed information about the selected bone. The selection mechanism includes visual highlighting, property display, and hierarchical navigation.

### Detail Panels
Enhanced detail panels display comprehensive bone information including:
- Bone name and identifier
- Position and rotation data
- Parent-child relationships
- Material and texture assignments
- Custom metadata and annotations

### Layer Toggle Controls
The designer agent extends `LayerToggle.tsx` to provide granular control over bone visibility, allowing users to:
- Show/hide individual bones
- Toggle entire bone groups or skeletal systems
- Manage layer visibility with persistence
- Filter bones by type or custom attributes

## Implementation Location

UI components are located in `dinolab/web/src/components/` and include:
- `BoneDetailPanel.tsx` - Extended for bone inspection and property display
- `LayerToggle.tsx` - Extended for visibility and layer management
- Supporting utility functions for bone hierarchy traversal and state management

## Handoff to Lane 08

The `ui_specification` output from this lane provides lane 08 (engineer) with all necessary information to implement API integration, including:
- Component prop interfaces and event signatures
- State management requirements
- Data flow patterns for bone selection and filtering
- Integration points with the rendering backend

This design-first approach ensures that the UI layer is thoroughly specified before engineering implementation begins, reducing iteration cycles and improving overall system coherence.
