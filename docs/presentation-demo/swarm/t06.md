# Swarm lane 06: 3D Rendering Pipeline Technician

## Overview

Lane 06 is the 3D rendering pipeline technician agent responsible for preparing render-ready assets by integrating skeletal rig data from lane 04 with muscle simulation data from lane 05. This lane transforms anatomical components into a cohesive 3D rendering configuration suitable for visualization and animation.

## Responsibilities

### Input Processing

The technician receives two primary inputs:

- **Rig Model** (from lane 04): Skeletal structure with bone hierarchy, joint constraints, and transformation matrices
- **Muscle Data** (from lane 05): Muscle fiber definitions, attachment points, deformation parameters, and simulation results

### Pipeline Operations

1. **Geometry Integration**: Merges skeletal mesh with muscle deformation data to create unified 3D geometry
2. **Material Assignment**: Applies appropriate shaders and textures to different anatomical components (bone, muscle, fascia)
3. **Animation Parameter Configuration**: Establishes keyframe data, blend shapes, and animation controllers
4. **Performance Optimization**: Reduces polygon count and optimizes draw calls for real-time rendering

### Output Generation

The technician produces a `render_ready_asset` containing:

- **Geometry**: Optimized mesh with vertex attributes (position, normal, UV, weights)
- **Materials**: Shader definitions and texture references
- **Animation Parameters**: Skeletal animation data, muscle deformation curves, and blend shape targets
- **Metadata**: Asset versioning, component references, and rendering hints

## Integration Points

### Upstream Dependencies

- **Lane 04 (Rigger)**: Provides skeletal structure and bone hierarchy
- **Lane 05 (Muscle Simulator)**: Provides muscle deformation and simulation data

### Downstream Consumers

- **Lane 07 (Designer)**: Uses render-ready assets for scene composition and lighting design
- **Lane 23 (Photographer)**: Consumes assets for final rendering and image capture
- **AnatomyViewer3D.tsx**: React component that displays the prepared 3D assets
- **skeleton3dCustomRigs.ts**: Utility module for custom rig management and animation control

## Technical Specifications

### Rendering Configuration Schema

The rendering configuration includes:

```
render_config = {
  geometry: {
    meshes: [{ vertices, normals, uvs, weights, indices }],
    skeleton: { bones, joints, constraints }
  },
  materials: {
    shaders: [{ name, vertexProgram, fragmentProgram }],
    textures: [{ type, path, format }]
  },
  animation: {
    keyframes: [{ time, bone_transforms, muscle_deforms }],
    blendShapes: [{ name, targets }],
    controllers: [{ type, parameters }]
  }
}
```

## Quality Assurance

Before handoff to downstream lanes, the technician validates:

- All muscle deformations align with skeletal movement
- Geometry remains manifold and free of self-intersections
- Animation parameters produce smooth, artifact-free motion
- Asset file sizes meet performance budgets
- All references to upstream data are correctly resolved

## Status and Iteration

The technician operates iteratively, receiving feedback from designers and photographers to refine rendering parameters, optimize performance, and improve visual quality until the asset meets production standards.
