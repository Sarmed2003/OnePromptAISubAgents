# Swarm lane 31: Data Integrity Auditor

## Overview

Lane 31 is the Data Integrity Auditor agent within the swarm architecture. This lane verifies data integrity checksums across all DynamoDB records sourced from upstream lanes to ensure consistency and detect corruption or tampering.

## Responsibilities

The Auditor agent in lane 31 performs comprehensive integrity verification across four critical data sources:

- **Lane 01 (Fossil Data)**: Validates checksums for paleontological records, species classifications, and temporal metadata stored in DynamoDB.
- **Lane 03 (Measurements)**: Verifies integrity of quantitative measurement data including dimensions, weights, and derived metrics.
- **Lane 22 (Conservation Logs)**: Audits preservation and conservation activity records to ensure audit trail completeness and authenticity.
- **Lane 23 (Images)**: Validates checksums for image metadata and references to ensure visual asset integrity.

## Output

The lane produces an `integrity_report` containing:

- **Hash Mismatches**: Detailed list of records where computed checksums do not match stored values, including record identifiers, expected hashes, and computed hashes.
- **Recovery Recommendations**: Actionable guidance for each mismatch, such as re-validation from source, manual review, or data restoration procedures.
- **Summary Statistics**: Overall integrity status, percentage of records verified, and severity classification of detected issues.

## Integration

The integrity report generated by lane 31 feeds directly into lane 32 (Director) for publication sign-off. The Director uses integrity findings to determine whether data is safe for publication or requires remediation before release.

## Technical Implementation

The Auditor agent:

1. Queries DynamoDB tables corresponding to each upstream lane
2. Computes checksums using consistent hashing algorithms (SHA-256)
3. Compares computed values against stored integrity fields
4. Categorizes mismatches by severity and data type
5. Generates recovery recommendations based on mismatch patterns
6. Outputs structured report for downstream consumption
