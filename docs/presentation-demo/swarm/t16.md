# Swarm lane 16: Geologist Agent

## Overview

Lane 16 implements a specialized geologist agent within the swarm architecture that correlates fossil age estimates using radiometric and stratigraphic data sourced from lane 15 (provenance). This agent synthesizes chronological information to produce comprehensive age correlation reports with confidence intervals and geological period assignments.

## Contract

**Input**: `provenance_record` — radiometric dating results and stratigraphic context from lane 15

**Output**: `age_metadata` — age correlation report containing:
- Estimated age ranges with confidence intervals
- Geological period classifications
- Dating method confidence scores
- Cross-validation between radiometric and stratigraphic approaches

## Responsibilities

The geologist agent performs the following core functions:

1. **Age Correlation**: Synthesizes multiple dating methodologies (radiometric, stratigraphic, biostratigraphic) to establish reliable age estimates for fossils and geological samples.

2. **Confidence Assessment**: Calculates confidence intervals around age estimates based on measurement uncertainty, sample quality, and agreement between independent dating methods.

3. **Period Assignment**: Maps calculated ages to established geological timescale periods (Hadean, Archean, Proterozoic, Phanerozoic subdivisions, etc.) with appropriate precision levels.

4. **Quality Control**: Validates age estimates against known geological constraints, stratigraphy, and peer-reviewed reference datasets to ensure scientific accuracy.

## Data Flow

- **Upstream (Lane 15)**: Receives provenance data including radiometric measurements, stratigraphic position, and associated metadata
- **Downstream (Lane 17)**: Feeds age_metadata to the historian agent for exhibition timeline construction
- **Downstream (Lane 21)**: Provides age context to the statistician for population diversity analysis across temporal intervals

## Implementation Notes

The geologist agent integrates established geochronological standards and maintains compatibility with the broader swarm architecture through standardized data contracts and error handling protocols.
